01 — Khái niệm

PST là gì?

Persistent Segment Tree (PST) — hay còn gọi là Cây phân đoạn bền vững — là một biến thể của Segment Tree truyền thống cho phép lưu trữ nhiều phiên bản của cây sau mỗi thao tác cập nhật. Thay vì ghi đè node cũ, PST tạo node mới chỉ trên đường đi từ gốc đến lá.

💡 Trực giác

Hãy tưởng tượng Segment Tree thông thường như một cuốn sổ — mỗi lần sửa, bạn xóa và viết lại. PST giống như sao chép cuốn sổ trước khi sửa, nhưng thông minh hơn: nó chỉ sao chép các trang có thay đổi, chia sẻ những trang không đổi với phiên bản cũ.

🌲

Segment Tree gốc

Mỗi update ghi đè O(log n) node. Không lưu lịch sử. Chỉ có 1 phiên bản tại mọi thời điểm.

🔀

Path Copying

Kỹ thuật cốt lõi của PST. Chỉ tạo mới các node trên đường đi root→leaf, tái sử dụng phần còn lại.

📸

Snapshot / Version

Mỗi update tạo ra root mới. Lưu mảng root[] → truy cập phiên bản bất kỳ trong O(log n).

🧮

Không gian

Mỗi update tạo thêm O(log n) node. Tổng: O(n log n). Cần cấp phát node bằng pool tĩnh.

02 — Xây dựng

Các bước triển khai

  1. Định nghĩa cấu trúc Node

    Mỗi node lưu val (tổng/min/max...), con trái l, con phải r. Dùng pool cấp phát tĩnh thay cho new để tránh TLE.

  2. Build cây phiên bản 0

    Hàm build(l, r) đệ quy giống Segment Tree thông thường. Trả về index node trong pool. Lưu vào root[0].

  3. Hàm Update — tạo phiên bản mới

    Hàm update(prev, l, r, pos, val) nhận root phiên bản cũ. Đi từ gốc đến lá theo pos, tạo node mới trên đường đi, copy con còn lại từ phiên bản cũ. Trả về root mới → lưu root[i].

  4. Hàm Query — truy vấn phiên bản bất kỳ

    Hàm query(node, l, r, ql, qr) hoạt động y hệt Segment Tree thông thường, nhưng bắt đầu từ root[version] tùy chọn.

  5. Ứng dụng đặc biệt — Merge hai cây

    Với bài toán offline hoặc đếm trên đoạn: dùng PST trên mảng sau khi nén tọa độ (coordinate compression). Query = query(root[r]) - query(root[l-1]).

⚠️ Lưu ý quan trọng

Số node tối đa cần cấp phát: (n + q) * 40 là an toàn với n, q ≤ 10⁵. Đừng dùng new Node() vì sẽ rất chậm — luôn dùng pool tĩnh với biến đếm cnt.

03 — Implementation

Code C++ đầy đủ

Template PST tổng quát với ví dụ bài toán K-th smallest trên đoạn [l, r] — một trong những ứng dụng kinh điển nhất.

pst_template.cpp 
// ═══════════════════════════════════════════════
//  Persistent Segment Tree — Template C++
//  Bài: K-th smallest trên đoạn [l, r]
// ═══════════════════════════════════════════════
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 1e5 + 5;
const int MAXLOG = 40;   // log2(MAXN) * constant

// ───── Node Pool ─────
struct Node {
    int l, r;   // index con trái, con phải trong pool
    int val;    // tổng / đếm / ...
} tree[MAXN * MAXLOG];

int cnt = 0;          // số node đã dùng
int root[MAXN];      // root[i] = root phiên bản thứ i

// ───── Tạo node mới ─────
int newNode() {
    ++cnt;
    tree[cnt] = {0, 0, 0};
    return cnt;
}

// ───── Build phiên bản 0 ─────
int build(int l, int r) {
    int node = newNode();
    if (l == r) {
        tree[node].val = 0; // khởi tạo lá
        return node;
    }
    int mid = (l + r) >> 1;
    tree[node].l = build(l, mid);
    tree[node].r = build(mid+1, r);
    tree[node].val = tree[tree[node].l].val
                   + tree[tree[node].r].val;
    return node;
}

// ───── Update: tạo phiên bản mới ─────
// prev  : root của phiên bản cũ
// pos   : vị trí cần cập nhật (đã nén tọa độ)
// delta : thay đổi (+1 khi thêm phần tử)
int update(int prev, int l, int r, int pos, int delta) {
    int node = newNode();
    tree[node] = tree[prev]; // copy từ phiên bản cũ
    tree[node].val += delta;

    if (l == r) return node; // đã đến lá

    int mid = (l + r) >> 1;
    if (pos <= mid)
        // đi trái → tạo node trái mới, giữ phải cũ
        tree[node].l = update(tree[prev].l, l, mid, pos, delta);
    else
        // đi phải → tạo node phải mới, giữ trái cũ
        tree[node].r = update(tree[prev].r, mid+1, r, pos, delta);

    return node;
}

// ───── Query: K-th smallest trên [ql, qr] ─────
// u = root[l-1], v = root[r]  (chênh lệch = đoạn [l,r])
// k : thứ tự cần tìm
int kth(int u, int v, int l, int r, int k) {
    if (l == r) return l; // tìm thấy
    int mid = (l + r) >> 1;
    // số phần tử trong con trái của đoạn [ql,qr]
    int leftCount = tree[tree[v].l].val
                   - tree[tree[u].l].val;
    if (k <= leftCount)
        return kth(tree[u].l, tree[v].l, l, mid, k);
    else
        return kth(tree[u].r, tree[v].r, mid+1, r, k-leftCount);
}

// ───── Main ─────
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int n, q;
    cin >> n >> q;

    // Đọc mảng + nén tọa độ
    vector<int> a(n), sorted;
    for (auto& x : a) { cin >> x; sorted.push_back(x); }
    sort(sorted.begin(), sorted.end());
    sorted.erase(unique(sorted.begin(), sorted.end()), sorted.end());
    int m = sorted.size(); // số giá trị phân biệt

    auto compress = [&](int x) {
        return lower_bound(sorted.begin(), sorted.end(), x)
               - sorted.begin() + 1; // 1-indexed
    };

    // Build phiên bản 0 (rỗng)
    root[0] = build(1, m);

    // Thêm từng phần tử → tạo phiên bản mới
    for (int i = 1; i <= n; i++) {
        int pos = compress(a[i-1]);
        root[i] = update(root[i-1], 1, m, pos, 1);
    }

    // Xử lý query
    while (q--) {
        int l, r, k;
        cin >> l >> r >> k;
        // Phần tử thứ k nhỏ nhất trong a[l..r]
        int ans = kth(root[l-1], root[r], 1, m, k);
        cout << sorted[ans-1] << '\n';
    }
    return 0;
}

🔍 Giải thích hàm update

update_explained.cpp 
int update(int prev, int l, int r, int pos, int delta) {
    int node = newNode();      // (1) Tạo node mới (không xóa node cũ!)
    tree[node] = tree[prev];  // (2) Copy toàn bộ từ phiên bản cũ
    tree[node].val += delta;  // (3) Cập nhật giá trị node này

    if (l == r) return node;   // (4) Đến lá → kết thúc

    int mid = (l + r) >> 1;
    if (pos <= mid)
        // (5a) pos ở nửa trái:
        //  - Con trái → đệ quy tạo mới
        //  - Con phải → giữ nguyên từ prev (chia sẻ!)
        tree[node].l = update(tree[prev].l, l, mid, pos, delta);
    else
        // (5b) pos ở nửa phải:
        //  - Con phải → đệ quy tạo mới
        //  - Con trái → giữ nguyên từ prev (chia sẻ!)
        tree[node].r = update(tree[prev].r, mid+1, r, pos, delta);

    return node;  // (6) Trả về root của subtree mới
}
04 — Phân tích

Độ phức tạp & So sánh

Thao tác Segment Tree thường Persistent Segment Tree
Build O(n) O(n)
Update một điểm O(log n) O(log n) — tạo thêm log n node
Query đoạn O(log n) O(log n) — trên bất kỳ phiên bản
Bộ nhớ tổng O(n) O((n + q) log n)
Truy cập phiên bản cũ ❌ Không thể ✅ O(1) — qua root[v]
✅ Tóm tắt chi phí

Mỗi thao tác update tốn thêm đúng ⌊log₂ n⌋ + 1 node. Với n = 10⁵ và q = 10⁵ truy vấn, cần ~4 triệu node — hoàn toàn khả thi với pool tĩnh 4–5 triệu phần tử.

🔄 Rollback — hoàn tác update
🌳 HLD + PST trên cây
📅 Segment Tree theo thời gian
🗺️ Offline range query

⚡ Merge Sort Tree thay thế

SPOJ MKTHNUM — K-th Number (bài kinh điển nhất). CF 786C — Till I Collapse. CF 813F — Bipartite Classifier. VNOI TREAP — PST kết hợp LCA. Luyện tập các bài này sẽ thành thạo PST hoàn toàn.

🔄 Rollback — Undo thao tác

Vì PST lưu tất cả phiên bản, rollback về trạng thái k bước trước cực kỳ đơn giản:

rollback.cpp 
// Undo k bước — chỉ cần con trỏ!
int current_version = n;

// Rollback về k bước trước:
auto rollback = [&](int k) {
    current_version -= k;  // O(1) !
};

// Query trên trạng thái hiện tại:
auto query_current = [&](int l, int r, int k) {
    return kth(root[l-1], root[r], 1, m, k);
    // Sử dụng root[current_version] thay vì root[n]
};
06 — Cạm bẫy

Lỗi thường gặp

💥

Pool không đủ lớn

Quên nhân với hằng số. Công thức an toàn: (n + q) * 40. Nếu MLE thì giảm constant.

🐛

Quên nén tọa độ

Với bài k-th smallest, giá trị a[i] có thể đến 10⁹ — phải nén xuống [1, m] trước khi update.

⚠️

Off-by-one index

Root version 0 thường là cây rỗng. Update phần tử i tạo root[i]. Query đoạn [l,r] dùng root[l-1] và root[r].

🔧

Dùng new thay pool

Dynamic allocation rất chậm với 10⁶ node. Luôn dùng mảng tĩnh + biến đếm cnt.
